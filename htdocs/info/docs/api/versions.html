Ensembl Genomes Versions

This page uses data release 1, software release 52 as a version example. We recommend always using the latest versions of code and data.

Ensembl Versioning Scheme

Each Ensembl release has an integer version number associated with it which is used to identify the correct versions of the API, web code and databases that make up that release. The idea is that components with the same release version should work together - i.e. any web site built with 'version 52' API and web code, and 'version 52' databases should work correctly.

Ensembl Genomes Versioning Scheme

Each Ensembl Genomes release has two integer numbers associated with it. One is used to identify the release of Ensembl Genomes data (starting from 1 for the first release of Ensembl Genomes). The second number is the Ensembl Version number, and is used to identify the version of the database schemas, and the associated version of the Ensembl API that can be used to access the databases. We will endeavour to make Ensembl Genomes databases available through the same version of the API as is used to access Ensembl databases (for example, updating the schemas of existing data releases if it is not practical to synchronise an Ensembl Genomes data release with a new release of Ensembl software).

The BioMart code has its own versioning scheme, since it is independent of the Ensembl code - see the BioMart website for information on using the code separately, e.g. with the MartJ Java interface.

Archive! sites

Previous releases of Ensembl Genomes are internally archived and are currently unavailable for external users. We are working towards providing this service.

Ensembl Genomes Code Versioning

For the APIs and the web code, a CVS branch (essentially a named snapshot of the code) is made for each release and named with the release version number. The Perl API and web code can be checked out of CVS with the tag "branch-ensembl-versionnumber", e.g. branch-ensembl-52.

Sometimes alternations are made to the Ensembl codebase which cannot be introduced into these ensembl branches. Because of this Ensembl Genomes specific branches have been created under "branch-ensemblgenomes-egversion-ensemblversion". All branches created in this manner are compatible with the advertised version of Ensembl. If you are using just the API then we recommend using these branches to ensure compatibility.

Ensembl and Ensembl Genomes Data Versioning

The Ensembl database names have the format: 

<species><database type><release number><data version>

As an example, the name of the Human Core database for the Ensembl 32 release would be: 

homo_sapiens_core_32_35e

A letter suffix to the data version indicates a change in data without a change in assembly; e.g. a new gene build.

Ensembl Genomes database names have the format: 

<species><database type><data release number>Ensembl (software) release number><data version>

As an example, the name of the Plasmodium knowlesi Core database for the 1st release of Ensembl Genomes, utilising the Ensembl 52 schema and API, would be: 

plasmodium_knowlesi_core_1_52_1a

Bacterial species are grouped in collections sharing a single database. These databases have the format: 

<collection description><database type><data release number>Ensembl (software) release number><data version>
As some examples, the names of the Core databases holding genomes from the genii Escherichia and Shigella, and the database holding data for Mycobacterium species, are as follows: 
escherichia_shigella_collection_core_1_52_1a

mycobacterium_collection_core_1_52_1a

Individual species can still be accessed by species name through the use of the Ensembl Registry, making the grouping of genomes in collections transparent to users of the Ensembl API. See this page this page for more information about how to use the Ensembl Registry.

BioMart Versioning

BioMart will branch less frequently than Ensembl, and the release number bears no relation to that of Ensembl.

The code can be checked out of CVS - see the installation instructions for the CVS tag format.
